2. How would you use an interoperability standard to express these data?

When I was with Aprima using the HL7 standard was preferred that way when a message from our EHR was sent to the vaccine registry here in Texas. It was one that could easily be interpreted. There was only 1 way to interpret an ORM. Each ORM had a segment and a name as defined by HL7. Looking at the NDDF data in the treatments extract leading zeros could be an issue for standardization as the NDDF database uses leading zeros. I would prefer to include the leading zeros so the standard is the same no matter if the data is incoming or outgoing. Once I have this, NLM uses a CUI that I would like to join on for a more widely recognized identifier that could easily be used no matter what system is used. This part was not too bad. Python is quite similar to sql with maybe a bit of c# once you get the syntax down.
